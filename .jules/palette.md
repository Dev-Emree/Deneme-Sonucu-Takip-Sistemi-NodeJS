## 2024-05-24 - Accessible Non-Semantic Accordions
**Learning:** This app extensively uses `<div>` and `<li>` elements for interactive accordions (e.g., in exam listings) instead of semantic `<details>`/`<summary>` or `<button>` elements. These elements lack built-in focusability and keyboard event handling, making them invisible and inaccessible to keyboard and screen reader users. Simply binding a click handler to a generic div leaves out a significant portion of users.
**Action:** When working with existing custom interactive elements like accordions or custom dropdowns built with non-semantic tags, always explicitly add `role="button"`, `tabindex="0"`, `aria-expanded`, and `aria-controls`. Bind `keydown` listeners for `Enter` and `Space` alongside `click`, ensuring `preventDefault()` is called on `Space` to stop unwanted page scrolling. Update the `aria-expanded` state dynamically in the event handler to reflect the current visual state.
